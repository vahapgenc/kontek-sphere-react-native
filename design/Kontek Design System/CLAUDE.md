# Kontek Design System — Mobile · project rules

## Capitalisation (HARD RULE)
Sentence case everywhere. A word's first letter may be a capital; every letter after it is lower-case. **Never set text in all-caps / UPPERCASE** — this includes eyebrows, labels, section headers, badges, nav/tab labels and headings — and never use Title Case. The **only** exception is a person's initials (e.g. an avatar showing "AL" or "MK"). Established acronyms that are inherently capitalised (iOS, VAB, BankID, CSS) are not "words" and are fine.

Practically: there is no `text-transform: uppercase` anywhere in this system. Don't reintroduce it, and don't hand-type words in caps for emphasis. Applies to every Kontek application and asset.

## Voice
Swedish-first, warm but professional, second person ("du/dig"). Reassure first, then concrete tabular specifics. Verbs on buttons. No emoji, no hype.

## Colour discipline
Green = positive / approve / save and the create (+). Red = destructive / irreversible only. Status meaning is always a text-colour + soft-bg pair. Don't invent hex values — use the tokens in `colors_and_type.css` / `mobile.css`.
