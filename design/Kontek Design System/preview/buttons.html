Buttons

Knappen bär vyns viktigaste åtgärd. Fyra typer bygger hierarki — en primär per vy, resten trappas ner. Skriv ett verb på knappen, aldrig ett substantiv.

Type variants

Four types establish hierarchy — primary (one per view, the main CTA), secondary (support actions), ghost (low-priority), transparent (minimal / inline). Height 48px (tap-safe on iOS & Android) · radius 12px · 8px gap.

Mobile shapes — block & large

On a phone the main action is a full-width block button; --lg (56px) is the prominent sticky-bottom CTA. --sm (40px) is for compact controls inside dense rows.

With icons — leading · trailing

Leading icon for action context (download, add); trailing icon for direction or disclosure (dropdown, external link).

Icon button — states & toolbar

Icon-only buttons for compact toolbars and dense layouts (edit, delete, refresh). Always pair with a tooltip for accessibility.

Action group — confirm / cancel

Dialog and modal footers: primary on the right for the main confirmation, secondary for cancel. One primary per dialog, right-aligned.

Destructive action

Destructive actions use the ghost or transparent type to avoid over-emphasising risky operations. Never place “Ta bort” immediately beside “Spara” — separate it spatially or behind a confirmation dialog.

Riktlinjer

Dos & don'ts

Do
  • Keep one clear primary action per screen; make it a full-width --block on the phone.
  • Put a verb on the label — the action, not the feature name.
  • Use --approve (green) for save/approve, --danger only for destructive.
Don't
  • Use red for save or approve.
  • Show two primary buttons competing on one screen.
  • Place “Ta bort” right beside “Spara” — separate it or gate it behind a confirm.
