Read the README.md file within this skill, and explore the other available files.
If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.
If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.

Quick orientation:
- `colors_and_type.css` is the single source of truth for tokens — always link or copy it, never hardcode hex.
- `ui_kits/kontek-next/` has reusable React/JSX components (`Sidebar`, `Topbar`, `Dashboard` and `<Icon>`); `index.html` shows them composed.
- `preview/*.html` are spec cards for colors, type, spacing and components.
- Brand essentials: dark green sidebar (#122121) + warm off-white canvas (#F4F3EE); Open Sans throughout (display + UI); primary action button is teal-green (#395F61, hover/press #052225, 2px #395F61 focus ring, disabled #E2E4E9/#9CA3AE); accent green (#61BC8F) carries the positive approve/save CTA (dark text); red (#E1462E) is for destructive actions only; Swedish, sentence-case, "du", no emoji.
- **Capitalisation (hard rule):** sentence case everywhere — a capital first letter, the rest lower-case. Never set words in all-caps/UPPERCASE (eyebrows, labels, section headers, headings all included) and never Title Case. The ONLY exception is a person's initials (e.g. avatar "AL"). This applies to every Kontek application and asset; there is no `text-transform: uppercase` in the system.
- **Mobile-first:** the system is tuned for the iOS + Android payroll app — one brand-led look on both (no Material/HIG split), 48px minimum touch targets, mobile type scale (display 36 / body 16), 12px button radius, 16px gutters. Phone-native components (app bar, tab bar, list rows, bottom sheet, FAB, banner/snackbar/toast, dialog, loading) are in `mobile.css`; the nav-pattern explorer is `ui_kits/kontek-next/mobile/index.html`.
- **The Sphere look (promoted prototype decisions):** every card-like surface uses ONE floating-card recipe — `box-shadow: var(--sh-1)`, 20px radius, transparent border, `--line-2` hairline dividers (never a visible border). The whole app sits on one gradient, `--app-bg` (no other decorative gradients). Pay/hero surfaces use the soft-mint `--mint-grad` (forest text), never a dark fill. The app bar is transparent at rest and frosts in on scroll (`--frost-bg` / `--frost-blur` / `--frost-line`). Action/nav icon tiles are solid `--tile-fill` (#053F22) with a white icon; soft chips are `--green-soft`/`--green-deep`; avatars sit on `--guide`. Mutations are optimistic with an Undo snackbar. Green = approve/save, red = destructive only, counts are circles (statuses are pills).
- **Badges (two distinct things):** a **status pill** (`<Badge>`) says *what state* — pill, `4px 10px`, 11px/600, optional leading dot in the text colour; colour is never hand-set, you pick a **tone by meaning** from the six (`ok`, `warn`, `info` (default), `danger`, `neutral`, `brand`) and use its matched text + `--*-bg` pair; dot = live item state, no dot = static tag/metadata; labels 1–2 words, sentence case, never truncated. A **count badge** (`.ds-tab__badge`) says *how many* — an 18px circle that grows to a pill for 2+ digits, white 11px number, 2px `--surface` ring, shown only when `> 0`; `--green` fill is for the **bottom-nav tabs only**, while the bell and section headers use the neutral dark-grey `--ink-3` variant (`.ds-tab__badge--neutral`). Never mix them; `danger` is a negative/failed state, NOT the destructive-action colour; an uncovered state maps to `neutral`. Full spec: `uploads/Sphere Badges - Spec.md`.
- **Card anatomy & dos/don'ts:** every spec card in `preview/` opens with a short Swedish intro (`.dg-h1` + `.dg-intro`, matching the desktop DS) and carries exactly ONE Dos & don'ts section (`.kdd`, styles in `preview/_guidance.css`) — what to do, what to avoid, plus a visual good/bad pair where it helps. Don't add a second do/don't block to a card. The same guidance is collected per area in `README.md`. Canonical source spec: `uploads/SPHERE Mobile App - Design Spec.md`.
